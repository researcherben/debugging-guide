Welcome! Start here
This guide assumes an omnipotent (in the practical sense) troubleshooting person and is not constrained by roles or rights.

Sources of a defect report:

  • user
  • support staff
  • vendor

This guide is intended for the above people

Additional stakeholders:

  • management
Phases:
  1. detect / observe

    Is this problem

    • recurring / intermittint
    • novel / first time

      How do you know this has been observed before?

    • on-going / degredation
  2. notify impacted people and initialize a report of the incident
  3. assess
    • What are the consequences?
    • What is the impact?
    • Urgency Triage: ignore, create workaround, or fix?
  4. Update impacted people and update the incident report
  5. isolate / identify cause
    1. come up with a theory
    2. design an experiment to test the theory
    3. collect data

    Document each experiment and result

  6. address / resolve / recover

    If this changes the system, update the model

  7. post mortem report
    • identify the change of process / design / training / use
    • identify the trigger of the initial event
    • identify lessons learned

    Share the post mortem report with all stakeholders
